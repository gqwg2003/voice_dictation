## Требования к безопасности библиотек

Для обеспечения безопасности проекта необходимо использовать следующие минимальные версии библиотек:

| Библиотека   | Минимальная версия | Текущая рекомендуемая версия |
|--------------|-------------------|-----------------------------|
| Qt           | 6.10.0            | 6.10.0                      |
| whisper.cpp  | 1.7.5             | 1.7.5                       |
| nlohmann/json| 3.12.0            | 3.12.0                      |

## Инструкции по обновлению библиотек

### Обновление Qt до версии 6.10.0

1. Скачайте Qt 6.10.0 с [официального сайта](https://www.qt.io/download)
2. Установите Qt, выбрав необходимые компоненты:
   - Qt 6.10.0 Base
   - Qt Multimedia
   - Qt Network
   - Qt Widgets
3. Обновите пути в вашей системе сборки:
   - CMake: `-DCMAKE_PREFIX_PATH=/path/to/qt/6.10.0`
   - Убедитесь, что при запуске Qt Creator используется правильная версия Qt

### Обновление whisper.cpp до версии 1.7.5

1. Клонируйте репозиторий whisper.cpp:
   ```
   git clone https://github.com/ggml-org/whisper.cpp.git
   cd whisper.cpp
   git checkout v1.7.5
   ```

2. Соберите библиотеку:
   ```
   mkdir build
   cd build
   cmake ..
   cmake --build . --config Release
   ```

3. Установите библиотеку:
   ```
   # Linux/macOS
   sudo cmake --install .
   
   # Windows (с правами администратора)
   cmake --install .
   ```

4. Обновите модели Whisper:
   ```
   # Windows (PowerShell)
   powershell -ExecutionPolicy Bypass -File .\scripts\download-whisper-models.ps1 base
   ```

### Обновление nlohmann/json до версии 3.12.0

Библиотека nlohmann/json загружается автоматически через CMake с использованием FetchContent. Это обеспечивает использование именно указанной в CMakeLists.txt версии 3.12.0.

## Исправленные уязвимости в обновленных версиях

### Qt 6.10.0
- Устранены уязвимости CVE-2023-32762 - предотвращение переполнения буфера в QNetworkAccessManager
- Исправлены уязвимости CVE-2023-34410 - устранение проблем при обработке сетевых соединений
- Повышена безопасность обработки медиаконтента и предотвращение атак через подмену входных данных

### whisper.cpp 1.7.5
- Добавлена поддержка no_speech_threshold для предотвращения генерации текста во время тишины, что может использоваться для атак через аудио
- Исправлены проблемы с обработкой памяти при обработке сложных аудиозаписей
- Улучшена безопасность потоковой обработки данных
- Оптимизирована производительность при минимизации использования ресурсов

### nlohmann/json 3.12.0
- Устранены уязвимости связанные с десериализацией сложных JSON-объектов
- Улучшена валидация входных данных для предотвращения инъекций
- Повышена устойчивость к отказу в обслуживании при обработке глубоко вложенных структур
- Исправлены проблемы с утечками памяти при обработке больших объемов данных

## Практики безопасного кода

### Проверка входных данных
- Все внешние данные должны проходить проверку перед использованием
- Используйте методы WhisperRecognitionService::setNoSpeechThreshold() для настройки чувствительности к тишине
- При обработке аудиопотока проверяйте размер и формат данных

### Сетевая безопасность
- Используйте только HTTPS для взаимодействия с API
- Избегайте хранения ключей API в открытом виде
- При использовании QNetworkAccessManager всегда проверяйте SSL-сертификаты

### Целостность данных
- Используйте контрольные суммы для проверки загруженных моделей
- Проверяйте корректность JSON-данных перед десериализацией
- Применяйте RAII для управления ресурсами

### Защита от внедрения кода
- Применяйте подготовленные запросы вместо прямой подстановки строк
- Избегайте использования функций типа eval() или system()
- При предоставлении пользовательской возможности загружать файлы, строго проверяйте их типы и содержимое

## Регулярные проверки безопасности

Мы рекомендуем регулярно проверять наличие обновлений библиотек и устанавливать их для поддержания безопасности проекта:

1. Подпишитесь на уведомления о релизах библиотек на GitHub
2. Регулярно проверяйте CVE-базы данных для используемых библиотек
3. Используйте инструменты статического анализа кода (например, Clang-Tidy, PVS-Studio)
4. Проводите динамический анализ с использованием таких инструментов как Valgrind или AddressSanitizer

## Обеспечение безопасности при разработке

### Непрерывная интеграция
- Настройте CI для автоматической проверки обновлений зависимостей
- Включите проверки безопасности в процесс сборки
- Используйте OWASP Dependency-Check для отслеживания уязвимостей

### Код-ревью
- Проводите регулярные код-ревью с акцентом на безопасность
- Используйте чеклист безопасности при проверке изменений
- Проверяйте новые зависимости на известные уязвимости

### Тестирование безопасности
- Включите тесты на проникновение в процесс разработки
- Проверяйте граничные случаи при обработке входных данных
- Тестируйте поведение приложения при некорректных входных данных
